# Phase 05 Plan 01 — Captura de Observações (Wave 1)

**Goal:** Os dois hooks de captura que fazem "100% das sessões gerarem observações". `observe-tool-use.sh` (PostToolUse) é o coletor leve; `observe-session-end.sh` (Stop) é o gatilho de avaliação (lição ECC: Stop, NÃO UserPromptSubmit — leveza). Mais o doc do layout `~/.ideiaos/observations/`, desenhado para sync multi-máquina (arquivos JSONL, sem DB).

**Por que Wave 1 e independente:** só cria arquivos NOVOS em `source/hooks/` + `docs/instincts/`. Zero overlap com 05-02 (que cria `source/skills/` + `docs/instincts/instincts-layout.md`, arquivo distinto). Os arquivos compartilhados (setup.sh, manifests, README, recall/extract) ficam TODOS na Wave 2 (05-03). Roda em paralelo com 05-02.

- Privacidade (memory-hygiene.md Regra 1): **NUNCA** logar conteúdo de arquivo, diff, comando bash completo ou valores que possam ser secrets. Só metadados: nome da tool, path relativo, sinal de sucesso/erro, timestamp, session_id.
- Hooks: bash + `/usr/bin/python3` (sem jq), `set -uo pipefail`, fail-silent `exit 0`.

## Tasks

### Task 1 — observe-tool-use.sh (PostToolUse, leve, fail-silent) + observe-session-end.sh (Stop)

`source/hooks/observe-tool-use.sh` — PostToolUse (matcher `Edit|Write|MultiEdit|Bash` no registro da Wave 2). Conteúdo INLINE (executor copia):

```bash
#!/usr/bin/env bash
# =============================================================================
# observe-tool-use.sh — IdeiaOS PostToolUse hook (Continuous Learning v2)
# SOURCE: IdeiaOS v2
#
# Acrescenta UMA linha JSON (JSONL) por evento relevante a
#   ~/.ideiaos/observations/<projeto-slug>/observations.jsonl
#
# Coleta APENAS metadados (memory-hygiene.md Regra 1):
#   ts, session_id, project, tool, file (path relativo, sem conteúdo),
#   ext, ok (sucesso/erro), bash_verb (1º token do comando, ex "npm", NUNCA args)
# NUNCA loga: conteúdo de arquivo, diff, comando bash completo, env, secrets.
#
# Requisitos: <100ms overhead, fail-silent (exit 0 sempre), cria dirs sob demanda,
#             não bloqueia o tool use. Sem jq — só /usr/bin/python3. set -uo pipefail.
# Entrada (stdin): JSON PostToolUse { session_id, cwd, tool_name, tool_input, tool_response }
# Saída: NENHUMA (exit 0 puro).
# =============================================================================
set -uo pipefail

INPUT="$(cat 2>/dev/null || echo '{}')"

# Parse e montagem da linha JSONL inteiramente em python3 (rápido, 1 processo).
LINE="$(/usr/bin/python3 -c '
import json, sys, os, re, datetime

try:
    d = json.load(sys.stdin)
except Exception:
    sys.exit(0)

sid = str(d.get("session_id", ""))[:64]
# Sanitiza session_id contra path traversal (mesma regra do strategic-compact)
if re.search(r"[/\\]|\.\.", sid):
    sys.exit(0)

cwd = d.get("cwd", "") or os.getcwd()
proj = re.sub(r"[^a-z0-9-]", "-", os.path.basename(cwd.rstrip("/")).lower())[:40].strip("-") or "project"

tool = str(d.get("tool_name", ""))[:32]
ti   = d.get("tool_input", {}) or {}
tr   = d.get("tool_response", {}) or {}

# file path: SÓ o path, relativo ao cwd se possível. NUNCA conteúdo.
fpath = ti.get("file_path") or ti.get("path") or ""
fpath = str(fpath)[:300]
rel = fpath
try:
    if fpath and cwd and fpath.startswith(cwd):
        rel = os.path.relpath(fpath, cwd)
except Exception:
    rel = fpath
ext = os.path.splitext(rel)[1].lstrip(".")[:12] if rel else ""

# bash_verb: APENAS o 1º token do comando (ex "npm", "git"), nunca os argumentos.
bash_verb = ""
if tool == "Bash":
    cmd = str(ti.get("command", "")).strip()
    bash_verb = re.split(r"\s+", cmd)[0][:24] if cmd else ""
    # descarta se parecer um path/secret embutido
    if "/" in bash_verb or "=" in bash_verb:
        bash_verb = os.path.basename(bash_verb).split("=")[0][:24]

# ok: sinal de sucesso (defensivo — formato de tool_response varia)
ok = True
try:
    if isinstance(tr, dict):
        if tr.get("is_error") or tr.get("error") or tr.get("interrupted"):
            ok = False
        st = tr.get("stderr") or ""
        if isinstance(st, str) and "error" in st.lower()[:200]:
            ok = False
except Exception:
    pass

rec = {
    "ts": datetime.datetime.now().isoformat(timespec="seconds"),
    "session_id": sid,
    "project": proj,
    "tool": tool,
    "file": rel,
    "ext": ext,
    "bash_verb": bash_verb,
    "ok": ok,
}
print(proj)                       # linha 1: slug do projeto (p/ o bash montar o path)
print(json.dumps(rec, ensure_ascii=False))   # linha 2: o registro JSONL
' <<< "$INPUT" 2>/dev/null)"

# Sem saída do python → nada a fazer (input malformado, session inválida, etc.)
[ -z "$LINE" ] && exit 0

PROJ="$(printf '%s\n' "$LINE" | sed -n '1p')"
REC="$(printf '%s\n' "$LINE" | sed -n '2p')"
[ -z "$PROJ" ] && exit 0
[ -z "$REC" ] && exit 0

OBS_DIR="$HOME/.ideiaos/observations/$PROJ"
mkdir -p "$OBS_DIR" 2>/dev/null || exit 0
printf '%s\n' "$REC" >> "$OBS_DIR/observations.jsonl" 2>/dev/null || true

exit 0
```

`source/hooks/observe-session-end.sh` — Stop (modelo: session-summary.sh, mas MUITO mais leve). Conteúdo INLINE:

```bash
#!/usr/bin/env bash
# =============================================================================
# observe-session-end.sh — IdeiaOS Stop hook (Continuous Learning v2)
# SOURCE: IdeiaOS v2
#
# Gatilho de AVALIAÇÃO de fim de sessão (lição ECC: usar Stop, não
# UserPromptSubmit — leveza). Acrescenta UM marcador "session_end" na
# observations.jsonl do projeto. NÃO dispara análise pesada aqui — apenas
# marca o fim; /instinct-analyze (skill, plan 05-02) é quem processa depois.
#
# Mesmo contrato de privacidade do observe-tool-use: só metadados.
# Sem jq — só /usr/bin/python3. set -uo pipefail. exit 0 puro.
# Entrada (stdin): JSON Stop { session_id, transcript_path, cwd }
# Saída: NENHUMA (exit 0).
# =============================================================================
set -uo pipefail

INPUT="$(cat 2>/dev/null || echo '{}')"

LINE="$(/usr/bin/python3 -c '
import json, sys, os, re, datetime
try:
    d = json.load(sys.stdin)
except Exception:
    sys.exit(0)
sid = str(d.get("session_id", ""))[:64]
if re.search(r"[/\\]|\.\.", sid):
    sys.exit(0)
cwd = d.get("cwd", "") or os.getcwd()
proj = re.sub(r"[^a-z0-9-]", "-", os.path.basename(cwd.rstrip("/")).lower())[:40].strip("-") or "project"
rec = {
    "ts": datetime.datetime.now().isoformat(timespec="seconds"),
    "session_id": sid,
    "project": proj,
    "tool": "session_end",
    "event": "session_end",
}
print(proj)
print(json.dumps(rec, ensure_ascii=False))
' <<< "$INPUT" 2>/dev/null)"

[ -z "$LINE" ] && exit 0
PROJ="$(printf '%s\n' "$LINE" | sed -n '1p')"
REC="$(printf '%s\n' "$LINE" | sed -n '2p')"
[ -z "$PROJ" ] && exit 0
[ -z "$REC" ] && exit 0

OBS_DIR="$HOME/.ideiaos/observations/$PROJ"
mkdir -p "$OBS_DIR" 2>/dev/null || exit 0
printf '%s\n' "$REC" >> "$OBS_DIR/observations.jsonl" 2>/dev/null || true

exit 0
```

### Task 2 — Harness de smoke test: test-observe-hooks.sh

`source/hooks/test-observe-hooks.sh` — segue o estilo do `source/hooks/test-hooks.sh` existente (bash, asserts, exit não-zero em falha). Usa um `HOME` temporário (`export HOME="$(mktemp -d)"`) para NÃO sujar o `~/.ideiaos/` real. Casos a cobrir:

1. **PostToolUse Edit** → input JSON `{session_id, cwd, tool_name:"Edit", tool_input:{file_path:"<cwd>/src/x.ts"}, tool_response:{}}` por stdin → asserta que `$HOME/.ideiaos/observations/<slug>/observations.jsonl` existe e tem 1 linha com `"tool": "Edit"`, `"ext": "ts"`, `"file": "src/x.ts"` (relativo).
2. **PostToolUse Bash** → `tool_name:"Bash", tool_input:{command:"npm run build --silent"}` → assertar linha com `"bash_verb": "npm"` e que a linha NÃO contém `"build"` nem `"--silent"` (privacidade: só o verbo).
3. **Privacidade — sem conteúdo** → input com `tool_input:{file_path:"...", content:"SENHA_SECRETA_123"}` → assertar que `SENHA_SECRETA_123` NÃO aparece em lugar nenhum da jsonl.
4. **Path traversal** → `session_id:"../../etc"` → assertar exit 0 e que NENHUM arquivo foi criado fora de `$HOME/.ideiaos/`.
5. **Input malformado** → stdin `"{ not json"` → assertar exit 0 sem erro e sem criar jsonl.
6. **Dirs sob demanda** → garantir que rodando com `$HOME/.ideiaos/` inexistente, o primeiro evento cria a árvore (já coberto por #1 começando de HOME limpo).
7. **Stop marker** → input Stop `{session_id, cwd}` no `observe-session-end.sh` → assertar linha com `"event": "session_end"`.
8. **Performance** → medir `observe-tool-use.sh` com `time`; assertar overhead razoável (informativo; aviso se > ~200ms em CI, não falha dura — o alvo de <100ms é em máquina do dev).

Fechar com um sumário `PASS/FAIL` e `exit 0` só se todos passarem.

## Notes

- **NÃO** registra os hooks em setup.sh / templates aqui — isso é shared-file (settings registration) e fica na Wave 2 (05-03), junto com manifests/README. Esta plan só PRODUZ os hooks em `source/hooks/`.
- O alvo de <100ms é por evento; o python3 single-shot mantém isso. Se o harness medir mais em CI lento, é informativo (a máquina do dev é o ambiente real).
- O `bash_verb` é deliberadamente conservador: só o 1º token, descartando qualquer coisa com `/` ou `=` (evita vazar paths de binários custom ou `VAR=valor cmd`). Em dúvida, prefere string vazia.
- `ok` é heurístico e defensivo — o formato de `tool_response` varia entre tools; nunca falha o hook por não conseguir determinar sucesso.
- Replicabilidade futura: o schema da linha é o contrato consumido por 05-02 (`/instinct-analyze` lê `tool`, `file`, `ext`, `bash_verb`, `ok`, `session_end`). Mudanças no schema exigem atualizar 05-02.
