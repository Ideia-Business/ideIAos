<task type="auto">
  <name>Task 2: Write the IdeiaOS standard statusline script</name>
  <files>source/statusline/ideiaos-statusline.sh</files>
  <action>
Create `source/statusline/ideiaos-statusline.sh`, an executable statusline command for Claude
Code's `statusLine` settings field. It reads ONE JSON object on stdin and prints ONE line.

Requirements:
  - Line 1: shebang `#!/usr/bin/env bash`. Line 2: `# SOURCE: IdeiaOS v2`. NEVER use `<!--`.
  - `set -uo pipefail` (NOT `-e` — a statusline must never abort with non-zero; always print
    something and `exit 0`).
  - Read all of stdin into a variable. Use `node -e` (node is a hard prerequisite in setup.sh,
    so it is safe) to parse JSON robustly — do NOT hand-parse JSON with grep/sed (fragile).
    Inside node, on ANY parse error, fall back to empty values so the script still prints a line.
  - Extract, with safe fallbacks:
      * model        ← `model.display_name`            (fallback: "claude")
      * current_dir  ← `workspace.current_dir`          (fallback: $PWD)
      * branch       ← `workspace.git_worktree.branch` if present, else
                       `git -C "$current_dir" rev-parse --abbrev-ref HEAD 2>/dev/null` (fallback: "—")
  - Context %: Claude Code does not pass a context-usage number on stdin reliably. Derive a
    best-effort token figure from `cost.total_tokens` if present and render it ONLY when > 0
    (e.g. "ctx ~12k"); otherwise omit the segment. Do NOT fabricate a percentage you cannot
    compute — omit rather than mislead. Document this choice in a comment.
  - GSD phase (optional segment): walk up from current_dir (max ~10 levels, stop at $HOME)
    looking for `.planning/STATE.md`. If found, extract the body line matching
    `^Phase:\s*(\d+)\s+of\s+(\d+)` and render "P{n}/{m}"; if the frontmatter `status:` line says
    the milestone is done you may render the status word instead. If no STATE.md, omit silently.
    (Mirror the walk-up logic in ~/.claude/hooks/gsd-statusline.js — reference, do not import.)
  - Compose the final single line, segments separated by `  ·  `, in this order:
      `<branch>  ·  <model>  ·  <basename current_dir>  ·  [P{n}/{m}]  ·  [ctx ~Nk]`
    Wrap each segment build so a failure in one segment never blocks the others.
  - End with `exit 0`.

Make it executable: `chmod +x source/statusline/ideiaos-statusline.sh`.
  </action>
