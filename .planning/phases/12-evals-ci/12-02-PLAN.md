# Fase 12 — Plan 02: GitHub Actions Workflow + Documentação

## Objetivo

Criar `.github/workflows/evals.yml` com dois jobs:
1. **`structural`** — sempre roda, sem API key, grátis: valida sintaxe bash dos scripts, frontmatter dos 22 casos, contagem >= 22, e `run-evals.sh --dry-run` exit 0. Falha bloqueia o PR.
2. **`llm-evals`** — roda quando `ANTHROPIC_API_KEY` secret existe OU via `workflow_dispatch`. Chama `run-evals.sh --ci`. Falha em invariante (`pass^k`) bloqueia o PR; falha em capacidade (`pass@k`) emite warning sem bloquear.

Atualizar `evals/README.md` com seção "CI/CD" documentando a política e o setup do secret.

## Tarefas

### Task 1 — R3-12: Criar .github/workflows/evals.yml

**Arquivos:** `.github/workflows/evals.yml`

Criar os diretórios `.github/` e `.github/workflows/` e o arquivo de workflow.

**Triggers:**
```yaml
on:
  push:
    branches: [work, main]
    paths:
      - 'source/**'
      - 'evals/**'
  pull_request:
    paths:
      - 'source/**'
      - 'evals/**'
  workflow_dispatch:
    inputs:
      run_llm:
        description: 'Rodar job LLM (requer ANTHROPIC_API_KEY secret)'
        required: false
        default: 'false'
        type: choice
        options: ['true', 'false']
```

**Job 1 — `structural`** (ubuntu-latest, sem secrets):

Steps:
1. `actions/checkout@v4`
2. **Bash syntax check:** `bash -n` em todos os scripts listados:
   - `evals/run-evals.sh`
   - `scripts/build-adapters.sh`
   - `scripts/hooks/observe-session-end.sh` (se existir — usar `[ -f path ] && bash -n path`)
   - Glob geral: `find scripts/ -name "*.sh" -exec bash -n {} \;` com erro em falha
3. **Contagem de casos:** `COUNT=$(ls evals/cases/EVAL-*.md 2>/dev/null | wc -l | tr -d ' ')` e `[ "$COUNT" -ge 22 ] || { echo "FALHA: apenas $COUNT casos (mínimo 22)"; exit 1; }`
4. **Validação de frontmatter dos casos:** script python3 inline que lê cada `evals/cases/EVAL-*.md`, extrai o bloco entre os delimitadores `---`, e verifica que os campos `id`, `title`, `source`, `mode`, `metric`, `k`, `severity` estão presentes. Se qualquer campo estiver ausente, imprime o arquivo e sai com exit 1. Usar `python3 -c` com heredoc — script <= 20 linhas.
5. **Dry-run do runner:** `bash evals/run-evals.sh --dry-run` deve retornar exit 0.
6. **Validação YAML do próprio workflow:** `python3 -c "import yaml, sys; yaml.safe_load(open('.github/workflows/evals.yml'))" || { echo "YAML inválido"; exit 1; }`

**Job 2 — `llm-evals`** (ubuntu-latest):

Condição de execução: `if: secrets.ANTHROPIC_API_KEY != '' || github.event_name == 'workflow_dispatch'`

Nota: no workflow YAML, a condição de secret em `if:` não é suportada diretamente para branches não-default. A forma correta é usar um step com `env` que verifica a variável:

```yaml
- name: Check API key disponível
  run: |
    if [ -z "$ANTHROPIC_API_KEY" ]; then
      echo "ANTHROPIC_API_KEY não configurado — pulando job LLM"
      echo "skip=true" >> $GITHUB_OUTPUT
    fi
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
  id: check_key
```

Steps do job:
1. `actions/checkout@v4`
2. **Instalar Claude CLI:** `npm install -g @anthropic-ai/claude-code` (necessário para `claude -p`). Adicionar `--quiet` para suprimir output. Verificar com `claude --version`.
3. **Check key** (step acima)
4. **Rodar evals com --ci:**
   ```yaml
   - name: Executar evals --ci
     if: steps.check_key.outputs.skip != 'true'
     run: bash evals/run-evals.sh --ci
     env:
       ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
   ```
   O `run-evals.sh --ci` já aplica a política: falha pass^k = exit 1 (GitHub Actions marca o job como falho e bloqueia o PR).

5. **Upload de resultados como artifact:**
   ```yaml
   - name: Upload resultados
     if: always()
     uses: actions/upload-artifact@v4
     with:
       name: eval-results-${{ github.run_id }}
       path: evals/results/
       if-no-files-found: ignore
   ```

**Configuração de branch protection** (não automatizável — documentar em README):
Para que o job `structural` bloqueie merges: no GitHub → Settings → Branches → Add rule para `main`: marcar "Require status checks to pass before merging" → selecionar `structural`.

**Verificação do arquivo criado:**
- `python3 -c "import yaml; yaml.safe_load(open('.github/workflows/evals.yml'))"` exit 0 (YAML válido)
- `grep -c "structural\|llm-evals" .github/workflows/evals.yml` >= 2

### Task 2 — R3-13: Atualizar evals/README.md com seção CI/CD

**Arquivo:** `evals/README.md`

Adicionar seção `## CI/CD — Execução Automática` após a seção "Como Executar" existente. Não alterar nenhuma seção existente.

Conteúdo da seção a adicionar:

```markdown
## CI/CD — Execução Automática

### Workflow: `.github/workflows/evals.yml`

A suíte executa automaticamente em push/PR para `source/**` ou `evals/**` via dois jobs:

#### Job `structural` (sempre ativo, sem custo de API)

Valida sem chamar LLM:
- Syntax bash em todos os scripts (`bash -n`)
- Contagem de casos >= 22
- Frontmatter completo em todos os `EVAL-*.md` (campos obrigatórios: id, title, source, mode, metric, k, severity)
- `run-evals.sh --dry-run` exit 0

**Bloqueia PR em falha.** Configurar como required check em Settings → Branches.

#### Job `llm-evals` (requer secret ou workflow_dispatch)

Roda `run-evals.sh --ci` com API key real. Política de saída:

| Tipo | Métrica | Comportamento em falha |
|------|---------|------------------------|
| Invariante de segurança/dados | pass^k | **exit 1 — bloqueia merge** |
| Capacidade de produtividade | pass@k | warning no log — não bloqueia |

Resultados salvos em `evals/results/YYYYMMDD-HHMM.jsonl` e disponíveis como artifact no run.

### Configurar ANTHROPIC_API_KEY como secret

1. No GitHub: **Settings → Secrets and variables → Actions → New repository secret**
2. Nome: `ANTHROPIC_API_KEY`
3. Valor: chave de API Anthropic (obtida em https://console.anthropic.com/settings/keys)

Sem o secret, o job `llm-evals` é pulado automaticamente (não falha o build). Para forçar execução manual: **Actions → Run workflow → `run_llm: true`**.

### Custo estimado por run

- Job `structural`: grátis (sem chamadas LLM), ~30s
- Job `llm-evals`: 22 casos × k=1 × ~2k tokens/caso ≈ 44k tokens input. Com Claude Sonnet: ~$0.13 por run. Considerar rodar apenas em PR para `main` (não em todo push para `work`).

### Política de bloqueio (R3-13)

- `pass^k` falhou → `run-evals.sh --ci` retorna exit 1 → job falha → PR bloqueado (se configurado como required check)
- `pass@k` falhou → mensagem `AVISO` no log → job retorna exit 0 → PR não bloqueado
- Sem API key → job `llm-evals` pulado → apenas `structural` bloqueia
```

**Verificação:**
- `grep "CI/CD" evals/README.md` retorna a seção
- `grep "ANTHROPIC_API_KEY" evals/README.md` retorna pelo menos 2 linhas (instruções + nome do secret)
- `grep "pass\^k\|pass@k" evals/README.md` retorna a tabela de política

**Critério de conclusão:** Seção "CI/CD" presente em README.md; instrução de configuração do secret clara e completa.
