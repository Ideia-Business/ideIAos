# Phase 03 Plan 02 — manifests/modules.json + Stack Detection

**Goal:** Criar `manifests/modules.json` catalogando todos os módulos IdeiaOS no formato ECC (id, kind, targets, deps, installStrategy) e adicionar função de detecção de stack ao setup.sh para instalar só rules relevantes.

## Context

O formato ECC para `modules.json` define cada módulo com:
- `id`: slug único (e.g., `hook-typecheck-on-edit`)
- `kind`: `hook` | `skill` | `agent` | `template` | `rule` | `context`
- `targets`: harnesses onde instala (`["claude", "cursor"]` ou só um)
- `deps`: lista de ids que este módulo depende
- `installStrategy`: `always` | `stack:STACK` | `manual`
- `description`: uma linha
- `source`: path relativo em source/

A detecção de stack analisa os arquivos do projeto alvo para identificar qual stack está presente e instalar rules relevantes.

### Task 2 — Função detect_stack() no setup.sh

Adicionar função `detect_stack()` ao setup.sh que retorna uma lista de stacks detectadas no projeto alvo (`$PROJECT_DIR`).

**Detecção por sinal:**

| Stack | Sinal |
|-------|-------|
| `typescript` | `tsconfig.json` ou `*.ts` em src/ |
| `react` | `react` em package.json dependencies |
| `nextjs` | `next` em package.json ou `next.config.*` |
| `supabase` | `supabase/` dir ou `@supabase/` em package.json |
| `lovable` | `lovable.config.*` ou `docs/lovable/` dir |
| `python` | `requirements.txt` ou `*.py` na raiz |
| `node` | `package.json` presente |

```bash
detect_stack() {
  local project_dir="${1:-$PWD}"
  local stacks=()
  
  [[ -f "$project_dir/package.json" ]] && stacks+=("node")
  [[ -f "$project_dir/tsconfig.json" ]] && stacks+=("typescript")
  
  if [[ -f "$project_dir/package.json" ]]; then
    grep -q '"react"' "$project_dir/package.json" 2>/dev/null && stacks+=("react")
    grep -q '"next"' "$project_dir/package.json" 2>/dev/null && stacks+=("nextjs")
    grep -q '"@supabase/' "$project_dir/package.json" 2>/dev/null && stacks+=("supabase")
  fi
  
  [[ -d "$project_dir/supabase" ]] && ! grep -q "supabase" <<< "${stacks[*]}" && stacks+=("supabase")
  [[ -f "$project_dir/lovable.config.js" || -d "$project_dir/docs/lovable" ]] && stacks+=("lovable")
  [[ -f "$project_dir/requirements.txt" || -f "$project_dir/pyproject.toml" ]] && stacks+=("python")
  
  echo "${stacks[*]}"
}
```

Integrar ao fluxo de setup: quando `detect_stack()` retorna stacks, logar quais rules serão instaladas (preparação para Phase 04+ que usa esse output).
