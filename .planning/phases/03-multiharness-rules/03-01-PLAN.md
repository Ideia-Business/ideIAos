# Phase 03 Plan 01 — source/ Migration

**Goal:** Criar estrutura `source/` como fonte única de verdade, mover todos os assets existentes e atualizar `setup.sh` para usar os novos paths — sem quebrar os 4 projetos-produto.

## Context

Atualmente `skills/`, `agents/`, `hooks/`, `templates/` ficam na raiz do IdeiaOS. O objetivo da Fase 03 é que `source/` seja a fonte de verdade e que `build-adapters.sh` (03-04) compile dela para cada harness. Esta migração é pré-requisito para os demais planos da Wave 1.

**Restrição crítica:** `setup.sh` usa variáveis `SKILLS_DIR`, `AGENTS_DIR`, etc. Elas devem apontar para `source/skills/`, `source/agents/`, etc. após esta mudança. Os 4 projetos-produto não chamam os paths diretamente — eles usam o setup.sh — então atualizar setup.sh é suficiente.

## Tasks

### Task 1 — Criar estrutura source/

Criar o diretório `source/` com subdiretórios espelhando a estrutura atual:

```
source/
├── skills/
├── agents/
├── hooks/
├── templates/
└── contexts/          ← novo (vazio, para Fase 07)
```

**Executar:**
```bash
mkdir -p source/{skills,agents,hooks,templates,contexts}
```

### Task 2 — Copiar assets para source/

Copiar (não mover) os assets existentes para source/. Manter os diretórios originais temporariamente como fallback (serão removidos no Wave 2 após verificação).

```bash
cp -r skills/. source/skills/
cp -r agents/. source/agents/
cp -r hooks/. source/hooks/
cp -r templates/. source/templates/
```

**Verificação:** `ls source/skills/ | wc -l` deve retornar o mesmo número que `ls skills/ | wc -l`.

### Task 3 — Atualizar setup.sh para usar source/

Localizar as definições de variáveis de diretório no setup.sh e atualizar para source/:

Buscar por `SKILLS_DIR`, `AGENTS_DIR`, `HOOKS_DIR`, `TEMPLATES_DIR` no setup.sh. Atualizar os valores para `"$IDEIAOS_DIR/source/skills"` etc.

Também verificar se há hardcoded paths como `$IDEIAOS_DIR/skills/` ou `$IDEIAOS_DIR/hooks/` no setup.sh e atualizar.

**Padrão de busca:**
```bash
grep -n "SKILLS_DIR\|AGENTS_DIR\|HOOKS_DIR\|TEMPLATES_DIR\|/skills/\|/agents/\|/hooks/\|/templates/" setup.sh
```

Atualizar todas as ocorrências relevantes para source/.

### Task 4 — Smoke test da migração

```bash
# Verificar que source/ tem tudo
echo "skills: $(ls source/skills/ | wc -l) items"
echo "agents: $(ls source/agents/ | wc -l) items"
echo "hooks: $(ls source/hooks/ | wc -l) items"
echo "templates: $(ls source/templates/ | wc -l) items"

# Syntax check do setup.sh
bash -n setup.sh && echo "setup.sh: OK"
```

### Task 5 — Commit

```
feat(03-01): source/ migration — fonte única de verdade

- source/skills/, agents/, hooks/, templates/, contexts/ criados
- Assets copiados de root → source/
- setup.sh atualizado: SKILLS_DIR et al → source/ paths
- Dirs originais mantidos como fallback (removidos na Wave 2)
```
