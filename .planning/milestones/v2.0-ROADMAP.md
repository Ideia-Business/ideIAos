## Visão geral

Transformação do IdeiaOS de um sistema focado em Claude+Cursor/stack-Lovable em um **canivete suíço universal de desenvolvimento com IA**: catálogo completo de skills/agents/rules para qualquer stack, arquitetura multi-harness com fonte única (`source/`), aprendizado contínuo automático via instincts, distribuição via marketplace de plugins, e segurança de agentes como infraestrutura.

Base: absorção curada do framework **ECC** (everything-claude-code, github.com/affaan-m/ECC, MIT) + receitas de 3 guias do criador. Resultado: sistema com 70 módulos catalogados, pipeline de quarentena obrigatório, 22 evals a partir de incidentes reais, e roteiro de 6 fases para v3.

## Realizações principais

1. **Pipeline de quarentena + catálogo ECC (27 módulos novos)** — `security/scan-absorbed.sh` com 4 checks (unicode invisível, HTML/JS payloads, comandos suspeitos, AgentShield) garante que nenhum conteúdo de terceiros entra sem scan. 27 módulos do ECC (13 agents + 14 skills) absorvidos com atribuição MIT e model routing explícito (haiku/sonnet/opus), levando o catálogo de 33 a 70 módulos.

2. **Arquitetura multi-harness com `source/` como fonte única** — `scripts/build-adapters.sh` compila `source/` para Claude e Cursor; dirs-raiz (`skills/`, `agents/`, `hooks/`, `templates/`) removidos completamente (superset verificado, `comm -23` vazio). `detect_stack()` detecta 7 stacks e instala apenas módulos relevantes por projeto.

3. **Continuous Learning v2 — instincts com confidence** — hooks PostToolUse+Stop coletam metadados de sessão em `~/.ideiaos/observations/<projeto>/observations.jsonl`; `/instinct-analyze` (haiku background) destila em instincts atômicos com score de confidence; `/evolve` promove instincts ≥0.7 para vault Obsidian ou `source/rules/` — sem segundo cérebro paralelo.

4. **Plugin marketplace + remoção de dirs-raiz** — `.claude-plugin/marketplace.json` com 3 sub-plugins (ideiaos-core/design-suite/lovable); `build-plugins.sh` gera `plugins/` idempotentemente de `source/`; máquina nova instala via `/plugin marketplace add Ideia-Business/IdeiaOS`.

5. **Suite de evals com 22 casos a partir de incidentes reais** — `evals/cases/EVAL-001..022` derivados de incidentes reais do ideiapartner (INC-3xx) e bugs do NFideia, usando metodologia pass@k/pass^k com runner `run-evals.sh`.

6. **Auditoria v3 — 15 gaps priorizados + 6 fases candidatas** — `docs/v3/v3-review.md` e `docs/v3/v3-roadmap.md`: 4 P1 (agents sem model/tools, instinct-loop manual, evals sem CI), 7 P2, 4 P3; 6 fases v3 derivadas dos gaps com esforço/valor mapeados.

## Resumo do milestone

**Desvios notáveis:**

- **rg → Python3 (Fase 02):** Os 3 primeiros checks de `scan-absorbed.sh` foram planejados com `rg`, mas output com U+200B e `<script>` corrompeu o rendering das ferramentas. Corrigido inline com Python3 — sem impacto em funcionalidade, mais portável.
- **README sync bloqueou commit da Fase 01:** `check-readme-sync.sh` exigiu que `test-hooks.sh` e `test-typecheck-on-edit.sh` também constassem no file tree. Auto-corrigido antes do push.
- **Commit acidental 08-01 + 08-02:** `docs/v3/` era diretório não rastreado; ao fazer `git add docs/v3/agents-audit.md`, o `token-economy-review.md` (criado concorrentemente pelo executor 08-02) entrou no mesmo commit. Não destrutivo.
- **Autosync race conditions:** o LaunchAgent de autosync capturou arquivos de trabalho intermediários em dois commits wip (7a4f54b, 2f9d25f) durante a sessão. O kill-switch de timeout 120s da Fase 02 mitiga futuros travamentos; a exclusão do `versions.lock` do `add -A` resolve a contaminação do pin GSD.
- **Frontmatter invertido em scan de quarentena (04-02):** WARN do AgentShield offline marcado como falso positivo — confirmado por inspeção manual e aceito.

**Decisões-chave:**

- Absorção ampla, não seletiva: todos os stacks no catálogo; seleção por projeto via manifests (2026-06-11)
- Quarentena obrigatória para 100% de conteúdo de terceiros; 36% das skills públicas têm prompt injection (AgentShield data)
- Atribuição MIT em todo conteúdo absorvido do ECC
- `source/` como fonte única → `build-adapters.sh` para cada harness; nunca editar artefatos gerados
- Skills-first: 84 commands legados do ECC não absorvidos (o próprio ECC migra para skills)
- Instincts desaguam no vault Obsidian (`/evolve`), não em segundo cérebro paralelo
- `setup.sh` permanece para bootstrap de máquina; plugin/marketplace distribui conteúdo versionável
- Contexts via `--system-prompt` (aliases CLI), não arquivos lidos por tool — autoridade de system prompt
- Model routing obrigatório: haiku (busca/worker), sonnet (default), opus (arquitetura/segurança)
- ssh/scp em `permissions.ask`, não deny — evita quebrar os 4 projetos-produto
- settings.json nunca modificado automaticamente por setup.sh (T-01-10 — snippets manuais)
- `AbandonProcessGroup false` no LaunchAgent: mata processos filhos git no timeout
- mgrep e typescript-LSP adiados para v3 — documentação-only nesta fase
- `run-evals.sh` manual por design — runner automático é gap G-04 (P1) para v3
- `slides` standalone candidata a aposentadoria em favor de `design-system` (G-12)

**Dívida técnica incorrida:**

- `ideiaos-checker.md` com inconsistência `name: setup-checker` vs filename — a corrigir em v3 agent-contracts (G-02)
- `strategic-compact` invoca subprocess Python3 a cada tool call — substituir por contador bash puro em v3 (G-06)
- `nc ` em `scan-absorbed.sh` gera falsos positivos em TypeScript (`.then(` → confunde nc) — regex a afinar em v3 (G-15)
- `banner-design` e `frontend-visual-loop` referenciam skills fora do modules.json (G-12, G-13)
- `/ideiaos-catalog` desatualizado (menciona 60 módulos, real é 70+) — atualizar em manifest-cleanup (G-14)
- Deny rules baseline em máquinas existentes: enforcement manual (G-10)

**Pendências diferidas para v3:**

- Scheduler automático para `/instinct-analyze` (G-03 — P1)
- CI/CD — `run-evals.sh` integrado em pipeline automático (G-04 — P1)
- Propagação em lote de setup.sh para projetos-alvo (G-09)
- Contexts verificados pelo `idea-doctor.sh` (G-11)
