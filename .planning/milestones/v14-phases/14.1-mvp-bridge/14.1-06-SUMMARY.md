## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Falso-positivo de entropia no bundle minificado do SPA (S5) e em paths/URLs**
- **Found during:** Task 2 (1ª execução do gate completo)
- **Issue:** O candidato de entropia inicial `[A-Za-z0-9_+/=.:-]{20,}` cruzava separadores de código/path/URL, mesclando identificadores JS (`dangerouslySetInnerHTML`, `Object.getOwnPropertyDescriptor`), refs git e o path do db em pseudo-tokens de alta entropia → S5 e S7 reprovavam o snapshot LEGÍTIMO (o modo de falha "gate vira ruído" que o doc 78 §1.2 antecipa).
- **Fix (por SHAPE, nunca por valor):** (a) candidato restrito a `[A-Za-z0-9_-]` (shape real de chave/token de API, não cruza `. : / =`); (b) exigir `>=1 dígito` (segredo random é digit-bearing; identificador minificado é word-like sem dígito — separação decisiva e verificada: todos os 43 falso-positivos de S5 tinham 0 dígitos, o veneno tinha 10); (c) allowlist de hex ≤64 e URL/.git-shape mantida. Verificado: S1/S2/S5/S7 = 0 falso-positivos; veneno (H=5.0) e sk- (14 dígitos) seguem pegos.
- **Files modified:** source/agentd/zeroleak-snapshot.sh
- **Commit:** 13b4ca6

**2. [Rule 1 - Bug] `set -euo pipefail` + grep sem match abortava o detector com falso-FAIL em S7**
- **Found during:** Task 2 (S7 reprovava sem mensagem de DETECTADO)
- **Issue:** Numa superfície sem token `[A-Za-z0-9_-]{20,}` (ex.: a linha de log do read.js), `grep -oE` sai 1; dentro de `ENTROPY_HITS=$(grep ... | awk ... | wc -l)` com `pipefail`, o pipeline herdava o exit 1 do grep e, sob `set -e`, abortava o detector inteiro → exit 1 fantasma (sem segredo real).
- **Fix:** `{ grep -oE ... || true; } | awk ...` — o grep nunca falha o pipeline.
- **Files modified:** source/agentd/zeroleak-snapshot.sh
- **Commit:** 13b4ca6

**3. [Rule 2 - Missing critical] Linha do `test-zeroleak.sh` no README (forçado pelo pre-commit `check-readme-sync.sh`)**
