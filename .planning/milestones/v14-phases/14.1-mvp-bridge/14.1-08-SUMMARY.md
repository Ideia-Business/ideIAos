## Deviations from Plan

### D1 — [Rule 1 - transcription bug in plan AC, not in code] A9 non-regression diff form

**Found during:** Task 1 (A9 proof).
**Issue:** The Task 1 acceptance criterion (line 83) and the VALIDATION.md A9 row write the non-regression diff as a **symmetric** cut at `━━━ Resumo ━━━` on BOTH sides. That cut includes §15 in the compared region. Since this task's entire purpose is to MODIFY §15, this symmetric diff can never be empty — it contradicts the AC's own stated intent ("só a §15 e a linha de contagem do Resumo mudam legitimamente").
**Resolution:** The verifier-reexecutable proof of "14 old sections byte-identical" is the symmetric cut at **`15) Cockpit`** (both sides `/15) Cockpit/,$d`), which compares sections 1–14 only and exits **0**. This matches the cited precedent's spirit (14.0-07 line 158 cuts the post side at `/15) Cockpit/,$d` to exclude the §15 region being worked on). Both diff forms are reported above. No code change needed — the §15 extension itself is correct; only the AC command literal is mis-transcribed.

### D2 — [Rule 1 - plan example uses wrong JSON type] §15 --json id is a string, not int

**Found during:** Task 1 (AC4) and Task 2 (A9).
**Issue:** The plan/VALIDATION examples assert `any(s["id"]==15 ...)` (int). The idea-doctor JSON sink emits `"id":"15"` (string) for ALL sections (pre-existing sink contract, lines 798/805 — unchanged by this plan). The int form fails `AssertionError`.
**Resolution:** Used `str(s["id"])=="15"` (satisfies the intent "section id==15 present"). This is a pre-existing sink contract, not a regression introduced here — §15 in 14.0-07 emitted a string id too. No code change.

### D3 — §15 read-model check emits WARN (not pass) on this machine

**Found during:** Task 1 verification.
**Issue:** The read-model holds a snapshot for `c706ac77d577` (Mac mini) but this machine's `machine_id` is `131fd55c7b65` (no row). Check (d) therefore warns "read-model sem snapshot de 131fd55c7b65".
**Resolution:** This is the **correct designed behavior** (graceful degradation, decision above) — WARN, never FAIL, so idea-doctor stays exit 0 (A9 green). On the Mac mini (where ingest runs against its own snapshot) the same check would `pass`. No fix needed.
