## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | Bash exit-code gates + Node assertions (no jest/vitest; the project's discipline is `antifragile-gates` exit-codes, not a JS test runner) |
| **Config file** | none — gates are scripts (`scripts/check-cockpit.sh`, `source/agentd/zeroleak-snapshot.sh`, `scripts/ttt-median.sh`, `scripts/test-zeroleak.sh`, `scripts/idea-doctor.sh`) + npm scripts in `apps/cockpit/package.json` |

| **Full suite command** | the per-criterion gate chain A1–A12 (below) |
| **Estimated runtime** | ~30–90s (Zero-Leak 7-surface + recorder re-derive dominate) |

---

## Two Verification Regimes (antifragile-gates)

- **F = artifact-of-file** — file written, build, test → `test -s` / exit-code is LAW, never the Read tool.
- **R = runtime/UI** — visual/interactive behavior with no exit-code → render + screenshot + a11y-tree
  (`frontend-visual-loop`), with an EXPLICIT criterion declared (never "looks right"). A7/A10 are R; everything else is F.

## Sampling Rate

- **After every task commit:** run the specific gate for that task (e.g., `bash scripts/check-cockpit.sh`, or the endpoint's curl assertion).
- **After every plan wave:** run the A-criteria touched by that wave, together.
  - Wave 1 (14.1-01): A5, A6 (endpoint + bind gate).
  - Wave 3 (14.1-02/03/04/05): A2, A7, A8, A10 (screens + ⌘K + Bridge median).
  - Wave 4 (14.1-06/07/08): A1, A3, A4, A9, A11, A12 (gates + closeout).
- **Before `/gsd-verify-work`:** A1–A12 all green by exit-code (A7/A10 add the regime-R screenshot with explicit criteria).
- **Max feedback latency:** < 90s.
