# Phase 14.1 Plan 04: Command Palette ⌘K — Authenticated browser→exec channel Summary

**The security-critical plan of v14.1: a `POST /command` channel that runs `spawnSync` on the operator's machine, closed by Origin+Host fail-closed auth + a per-boot ephemeral `X-Cockpit-Token`, a 6-verb typed default-deny enum, server-side arm-before-fire, a no-interpolation B5 daemon lookup-map, and Zero-Leak-scanned stdout — driven by a ⌘K `cmdk` palette that never exposes a production/cross-machine/@devops verb.**

## Accomplishments

### Task 1 — `read.js` POST /command (commit `cdec953`)

Added `handleCommand` (POST /command) + `handleCommandToken` (GET /command-token) to the existing loopback read server, in strict fail-closed order:

1. **Auth (S-01)** — `isTrustedOrigin()`: `Origin === 'http://127.0.0.1:5273'` AND `Host ∈ {127.0.0.1:PORT, localhost:PORT}` (Host check neutralizes DNS-rebinding). Then a per-boot `crypto.randomBytes(32)` token required in `X-Cockpit-Token`, compared with `timingSafeEqual`. The token is delivered to the SPA same-origin via the Origin-gated `GET /command-token`.
2. **Body hardened (S-02)** — `Content-Type: application/json` required (415); 4096-byte cap with stream-abort (413); `JSON.parse` in try/catch (400).
3. **Closed `VERBS` enum** (6 keys) — `const v = VERBS[body.verb]; if (!v) 403` (default-deny, references v14.4 gate).
4. **Arm-before-fire (server-side)** — `v.arm && body.confirmed !== true → 412`.
5. **B5 lookup-map (S-04)** — `body.daemon` indexes a fixed `DAEMON_MAP {envsync, refresh-ai-security, cockpit}`; the launchctl arg is built from the mapped label, never from `body.daemon`; `$UID` from `process.getuid()`. Unknown key → 403.
6. **Exec** — `spawnSync(cmd[0], cmd.slice(1), {cwd: ROOT})` — no shell.
7. **Zero-Leak stdout (S-03)** — combined stdout/stderr written to a 0600 tmpfile, scanned by `source/agentd/zeroleak-snapshot.sh` (exit 0 = clean); only clean output returned (truncated), otherwise redacted; fail-closed if the detector cannot run.

### Task 2 — ⌘K Command Palette (commit `1798f29`)

- `ui/command.tsx`: cmdk primitive ported from nfideia, adapted to the cockpit (inline `cn`, hand-rolled overlay instead of radix Dialog → no direct radix-dialog dep).
- `CommandPalette.tsx`: ⌘K/Ctrl+K listener → overlay with a fixed 6-verb allowlist (B1–B6, PT-BR labels). `onSelect` fetches the token same-origin from `/command-token`, POSTs `{verb[, confirmed][, daemon]}` with `X-Cockpit-Token` + `Content-Type: application/json`, and renders the result (exitCode/stdout/zeroleak) inline. B1/B3 require a 2-step UI confirmation before sending `confirmed:true` (defense-in-depth).
- `cmdk@^1.1.1` added as the only new direct dependency; `npm install` run; `npm run build` green.

## Deviations from Plan

**1. [Rule 3 - Blocking issue] Token delivery via Origin-gated `GET /command-token` instead of "inject into Vite HTML"**
- **Found during:** Task 1 design.
- **Issue:** The plan text says "injeta-lo na HTML que o server serve em :5273 same-origin." read.js serves :3073; the SPA HTML at :5273 is served by Vite (dev) / static dist (prod), which read.js does not control — it cannot inject a boot token into Vite's HTML.
- **Fix:** Implemented the security-equivalent: `GET /command-token` gated by the SAME `isTrustedOrigin()` (Origin+Host). The browser forces the real `Origin` header (SOP; JS cannot forge it cross-origin), so a malicious cross-origin tab receives 403 and cannot read the token — the identical invariant the plan requires ("a cross-origin tab must not be able to read it"). The SPA at :5273 reads it same-origin and replays it in `X-Cockpit-Token`.
- **Files modified:** apps/cockpit/server/read.js (handleCommandToken), apps/cockpit/src/components/CommandPalette.tsx (getToken).
- **Commit:** cdec953 (server), 1798f29 (client).

**2. [Note - dependency] cmdk pulls a radix-dialog subtree transitively**
- cmdk@1.1.1 depends on `@radix-ui/react-dialog` and friends. These appear in package-lock as transitive deps of the sanctioned `cmdk` — NOT as new direct cockpit deps. `package.json` adds only `cmdk`. To avoid adding a direct radix-dialog dep, `ui/command.tsx` hand-rolls the overlay (plain div + backdrop) rather than importing radix Dialog.

**3. [Safety] B1/B4 never fired**
- Per operational safety (autosync PAUSED for the whole phase), B1 (pause → would be idempotent but re-asserts pause) and B4 (force_sync → would sync) were NOT fired in any test. The positive 200 path used B6 (run_doctor, read-only). Autosync re-asserted PAUSADO after testing.
