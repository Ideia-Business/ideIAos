## O objetivo do bootstrap (o que estes proof-gates PROVAM)

Provar, **por exit-code e fail-closed**, que o mecanismo O2 + step-up funciona — **sem** habilitar
nenhuma mutação de produção nem comando cross-máquina real. O bootstrap é um **harness de prova
local**: gera chaves, assina, verifica, e prova que assinatura inválida / chave-não-pinada / papel
forjado / revogação-forjada / token-replay são **RECUSADOS**. Verde aqui = pré-condição de R-WP10
satisfeita; ainda assim a feature cross-máquina é uma fase posterior (e Q5 antes dela).

**Regime de verificação:** `antifragile-gates` — artefato-de-arquivo + exit-code binário, NUNCA o
Read tool. Cada prova materializa seu resultado e checa por exit-code. Sem mutação de produção.

## Não-objetivos (explícitos)

- ❌ Nenhuma chamada a API de provedor (Vercel/Railway/Supabase/GitHub) — bootstrap é cripto-local.
- ❌ Nenhuma habilitação de `rotate`/`revoke`/`deploy` real (continuam fora do allowlist).
- ❌ Nenhum comando cross-máquina real disparado — prova-se assinar/verificar, não executar.
- ❌ Q5 não é decidida aqui.
