## Implementation Notes

The 5 emitters (`pass`/`warn`/`fail`/`info`/`step`) were decorated in two parallel branches controlled by `JSON_MODE`:

- **ANSI mode** (default): existing `echo -e` calls untouched; arrays populated as side-effect
- **JSON mode** (`--json`): `echo -e ... >/dev/null` suppresses all ANSI; arrays still populated

The `step()` function parses `^([0-9]+)\)[[:space:]]*(.*)` via `[[ =~ ]]` to extract section id/title. The "Resumo" section (no number) never empilhates — it feeds `summary` directly from PASS/WARN/FAIL counters.

The JSON sink iterates with `while [ "$_i" -lt "$_sec_count" ]` loops (not `for x in "${ARR[@]}"`) to avoid the bash 3.2 `set -u` unbound-variable abort on empty arrays.
