## Phase Boundary

**v14.0 = Substrato + Espinha.** Torna o substrato IdeiaOS **federável** e faz nascer a SPA, **sem nenhuma UI de valor ainda** — é a fase de canalização. Entrega 5 grupos de capacidade + preparação + gates de fechamento:

1. `idea-doctor.sh --json` — flag nova (14 seções → JSON estruturado) **com fallback ANSI provado intocado** antes de qualquer consumidor depender do JSON.
2. Protocolo do ref `cockpit` — grava `snapshots/<machine_id>.json` em `refs/heads/cockpit` por `git hash-object`/`mktree`/`commit-tree`/`update-ref`, **sem tocar o working tree** (espelho de `push_planning_ref`).
3. `ideiaos-agentd` — coletor **read-only** (4º LaunchAgent `com.ideiaos.cockpit`, `StartInterval 900`) que normaliza só **metadata** (zero valor de segredo) e grava o snapshot no ref.
4. `console-ingest` → read-model **SQLite descartável** (`~/.ideiaos/console/read-model.db`, `rm && rebuild` reconstrói dos refs).
5. Harness de **Time-to-Truth** (baseline terminal N≥5 por jornada J1/J4/J2) + SPA scaffold (Vite/React/TS/Tailwind/shadcn black-gold) em loopback sem login.

**FORA de escopo (gated p/ v14.4):** qualquer verbo de mutação de produção ou cross-máquina (`rotate`/`revoke`/`deploy`/`git push`/`gh pr`). Esta fase é read-only quanto a produção. O agentd **só grava no ref `cockpit`** e **nunca** toca `main`.
</domain>

<decisions>
## Implementation Decisions (LOCKED — do usuário + ADR + spec)

### Arquitetura (ADR `docs/decisions/v14-cockpit-local-first-git-as-bus.md`, Aceito)
- **Local-first.** Nada de servidor central; cada máquina coleta o próprio estado.
- **Git-as-bus por REF (não working tree).** Ref órfão `cockpit` escrito por plumbing → o `git add -A` cego do autosync NUNCA captura (invariante A4: `git status --porcelain` vazio após gravar).
- **Control-plane (metadata), nunca cofre.** Gestão de chaves = METADATA (nome da var, present/expected, risk_tier, mtime). O **valor** de um segredo JAMAIS transita browser/LLM/DOM/rede/log/snapshot/ledger (`credential-isolation`). `ApiKey` **sem coluna `value`** — isolamento materializado no schema.
- **Defesa estrutural, não disciplinar.** Zero-Leak=0 é gate de release por exit-code, não "lembrar de não vazar".

### Nomes canônicos (BLUEPRINT D1)
- Ref de federação = **`cockpit`**. Daemon = **`ideiaos-agentd`**. LaunchAgent = **`com.ideiaos.cockpit`** (4º, irmão de envsync/gitautosync/refresh-ai-security, `StartInterval 900`).
- Onde docs 40/02 dizem `mission-control`, leia **`cockpit`**.

### Substrato
- `machine_id` = `sha256(IOPlatformUUID)[:12]` — derivado 1×, cacheado; **NUNCA hostname** (gotcha alias `192`↔`MacBook-Air-2`).
- Descoberta de produtos **dinâmica** (iterar `~/dev/*/.git`), classificar produto vs test-dir por heurística determinística — **nunca assumir N=5** (substrato real = 7 produtos, Jarvis 469 sessões).
- Dedup de `Machine` por `machine-aliases.json` curado; `gsd` por **string-equality** (nunca semver — `version-reset-migration-semver-trap`); ator por classe determinística (`wip: autosync`/`.local$`→autosync, `[bot]@`→bot, senão human).

### UI (espinha mínima)
- SPA **Vite + React + TS + Tailwind + shadcn/ui**, tema **black-gold OKLCH (`--brand-hue:75`)**, servida em `http://127.0.0.1` **loopback, sem login**. Só prova que a Espinha conecta substrato→UI (1 card de máquina).

- Reuso de componentes de SPA do nfideia **se existirem** (pattern-mapper verifica; senão greenfield).
- Detalhes de empacotamento Node do agentd/ingest (Node 18, sem deps pesadas; `better-sqlite3` aceitável p/ o ingest).

<specifics>
## Specific Ideas (do doc 72 — copiar como alvo concreto)

- **Schema JSON do doctor:** `ideiaos-doctor/v1` com `sections[]{id,titulo,status,counts,itens[]}` + `summary{ok,warn,fail,exit}`. `id`/`titulo` derivados do arg de `step()` via regex `^([0-9]+)\)[[:space:]]*(.*)$`. `summary.exit` idêntico ao exit-code ANSI.
- **Sequência exata do ref** (doc 72 §2): `hash-object -w` → base = árvore do `cockpit` atual (preserva outras máquinas via `ls-tree | grep -v "snapshots/$MID.json" ; append`) → `mktree` → `commit-tree` (`-p` se ref existe, órfão se não) → `update-ref`. **Nunca** `update-index`/índice temp.
- **SHAPE do snapshot:** `ideiaos-cockpit-snapshot/v1` com `machine_id, agentd_version, os_version, taken_epoch, daemons[], doctor{}, installed_versions{}, accounts[], projects[]{api_keys[]{var_name,present,expected,risk_tier,mtime_epoch} — sem value}`.
- **api_key DDL:** PK `(project_slug, var_name)`, `risk_tier CHECK IN ('critical','sensitive','low','none')`, **sem coluna `value`** — qualquer `ALTER` que a adicione = bloqueado (guard `PRAGMA table_info` no teste).
- **3 jornadas TtT:** J1 (frota saudável?), J4 (chave X existe + idade, nunca valor), J2 (pronto p/ tag? — soak ≥2 máquinas, span≥1d **sobre epochs gravados**, doctor verde, security re-selado). Baseline terminal N≥5, mediana.
</specifics>
