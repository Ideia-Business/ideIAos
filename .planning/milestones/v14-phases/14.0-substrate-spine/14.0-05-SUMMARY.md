## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] machine_id computation: shasum vs Node crypto divergência**
- **Found during:** Task 2 verification
- **Issue:** O plano e CONTEXT.md especificam `sha256(IOPlatformUUID)[:12]`. Shell usa `shasum -a 256` que adiciona `  -` ao output, resultando em `131fd55c7b65`. Node usa `crypto.createHash('sha256')` que gera hex puro, resultando em `c706ac77d577`. São canonicamente diferentes.
- **Fix:** Adotado o método Node (`crypto.createHash`) como canônico pois o agentd é Node e a consistência intra-agentd importa. A verificação dos acceptance criteria foi adaptada para usar o MID real do Node (`c706ac77d577`) em vez do MID shell.
- **Files modified:** source/agentd/collect.js (já usava crypto correto)
- **Impact:** MID do Mac mini = `c706ac77d577` (Node canonical). Documentado como decisão.

**2. [Rule 2 - Missing] os.hostname removido do fallback de machine_id**
- **Found during:** Task 1 acceptance criteria (`grep -c 'os.hostname' == 0`)
- **Issue:** Fallback original usava `os.hostname()` para gerar MID alternativo, violando a gotcha `192 <-> MacBook-Air-2`.
- **Fix:** Substituído por hash fixo de string `'unknown-iopuuid'` — opaco e consistente sem hostname.
- **Commit:** 3ddf283 (já incluído)
