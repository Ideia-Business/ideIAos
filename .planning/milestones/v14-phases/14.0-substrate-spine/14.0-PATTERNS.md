**Sequência exata do ref (doc 72 §2 — git plumbing puro, ZERO working tree):**
```bash
cockpit_write_snapshot() {            # args: MID  SNAPSHOT_JSON (caminho de arquivo)
  local MID="$1" snapshot_json="$2"
  # 1) blob para o json do snapshot
  local blob; blob=$(git hash-object -w "$snapshot_json")
  # 2) subárvore FLAT snapshots/ — entrada é SÓ "<mid>.json", SEM prefixo "snapshots/".
  #    APPEND: lê as entradas da subárvore snapshots do cockpit atual, descarta a desta máquina,
  #    adiciona a nova, e mktree do conjunto mesclado. ÓRFÃO: `existing` vazio → mktree só da nova.
  local existing; existing=$(git ls-tree refs/heads/cockpit:snapshots 2>/dev/null | grep -v $'\t'"$MID.json"'$' || true)
  local sub; sub=$(printf '%s\n100644 blob %s\t%s.json\n' "$existing" "$blob" "$MID" | grep -v '^$' | git mktree)
  # 3) árvore TOPO referenciando a subárvore sob o nome "snapshots"
  local top; top=$(printf '040000 tree %s\tsnapshots\n' "$sub" | git mktree)
  # 4) commit (parent = tip atual do cockpit se existir, senão órfão) e move o ref
  local parent; parent=$(git rev-parse -q --verify refs/heads/cockpit || true)
  local commit; commit=$(git commit-tree "$top" ${parent:+-p "$parent"} -m "cockpit: snapshot $MID")
  git update-ref refs/heads/cockpit "$commit"
}
cockpit_read_snapshot()  { git show "cockpit:snapshots/$1.json"; }            # arg: MID
cockpit_list_machines()  { git ls-tree --name-only cockpit snapshots/; }
```

**Adaptação / invariantes que o plano DEVE gatear:**
- **A4 (working tree intocado):** após `cockpit_write_snapshot` em repo /tmp, `[ -z "$(git status --porcelain)" ]` exit 0. NUNCA `git add`/`update-index`/índice temp (espelha learning `git-plumbing-partial-branch-overlay-sync`).
- **Árvore de DOIS níveis (mktree é um-nível-só):** `git mktree` rejeita path com barra (`snapshots/<mid>.json` → `fatal: path ... contains slash`, exit 128). Por isso constrói-se a subárvore FLAT `snapshots/` primeiro (entrada `<mid>.json`, sem prefixo) e depois a árvore-topo que a referencia sob o nome `snapshots`.
- **Preserva outras máquinas:** lista vem da **subárvore-alvo `cockpit:snapshots`** (não do índice); `ls-tree refs/heads/cockpit:snapshots | grep -v | append | mktree` (subárvore) → `mktree` (topo).
- **Idempotência por MID:** re-gravar `MID_A` substitui (não duplica) — `git ls-tree cockpit:snapshots | grep -c "${MID_A}.json"` == 1.
- **Órfão na 1ª vez:** `existing` vazio (sem ref) → subárvore só com a entrada nova; commit sem `-p` (`${parent:+-p ...}` expande para nada).
