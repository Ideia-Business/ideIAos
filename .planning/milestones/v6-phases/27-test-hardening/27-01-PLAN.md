--- test-deia-trigger.sh ---

HOOK: source/hooks/deia-trigger.sh
Tests run by piping JSON to the hook via stdin and capturing stdout + exit code.

Helper run_trigger(): takes a JSON string, runs `bash "$HOOK"` feeding it on stdin,
captures stdout; returns 0 always (hook always exits 0), stdout is what we check.

Test groups (use head() between groups):

1) Trigger detection — all Deia variants must produce additionalContext in stdout:
   - "Deia, faça algo" → stdout contains "additionalContext"
   - "deia, faça algo" → stdout contains "additionalContext" (lowercase)
   - "Déia, algo" → stdout contains "additionalContext" (accented)
   - "Deia: algo" → stdout contains "additionalContext" (colon variant)
   - "Deia algo" → stdout contains "additionalContext" (space, no punctuation)
   - stdout JSON must contain "IdeiaOS" (routing context present)

2) Non-trigger passthrough — hook emits NOTHING (empty stdout):
   - "Olá Deia, como vai" → empty (does NOT start with Deia)
   - "Bom dia, Deia" → empty (Deia not at start)
   - "Ideiadeia algo" → empty (Deia not at position 0)
   - "" (empty prompt) → empty (empty string input)

3) Edge cases:
   - Empty JSON {} → exit 0 (no crash), stdout empty (no prompt key)
   - Malformed input "not json at all" → exit 0 (no crash), stdout empty
   - JSON with null prompt: '{"prompt":null}' → exit 0, stdout empty
   - Leading whitespace: '{"prompt":"  Deia, algo"}' → stdout contains "additionalContext"
     (pattern allows leading whitespace per the grep in the hook)
