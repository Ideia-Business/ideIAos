Implementar o motor shell que torna o delta-spec real e determinístico (este é o núcleo NOVO do IdeiaOS — o OpenSpec faz isso em TypeScript no pacote npm; nós reimplementamos em shell para não depender do CLI). Cabeçalho de cada lib: `# SOURCE: OpenSpec MIT Fission-AI/OpenSpec | adapted: IdeiaOS v6` (conceito absorvido, código nativo).

`lib/spec-validate.sh <dir-da-change>` — GATE binário (estilo R6-01 `test -s`/exit code, não confiar em leitura):
- Exit 2 = erro de invocação (arg faltando / dir inexistente).
- Exit 1 = delta inválido. Exit 0 = válido.
- Validações: (a) todo `#### Cenário:` tem 4 hashtags exatos — rejeitar cenários com 3 `###` sob um requisito; (b) cada `### Requisito:` tem ≥1 `#### Cenário:`; (c) headers de requisito não duplicados dentro do mesmo arquivo de delta; (d) bloco `## MODIFICADO` exige o requisito completo (heurística: ≥1 `#### Cenário:` no bloco modificado); (e) `## REMOVIDO` exige `**Motivo**` e `**Migração**`. Reportar erros específicos com arquivo:linha. Para contagens, filtrar comentários/prosa antes de `grep -c` (`grep -v '^#' | grep -c` — higiene de gate).

`lib/spec-merge.sh <produto-root> <slug> [--dry-run] [--yes]`:
- PRIMEIRO chama `spec-validate.sh` na change; se exit≠0, ABORTA (não aplica nada).
- Resolve tokens canônicos a partir dos rótulos PT-BR do delta (ADICIONADO→ADDED, MODIFICADO→MODIFIED, REMOVIDO→REMOVED, RENOMEADO→RENAMED) para que o motor tenha um vocabulário interno estável.
- Aplica por capability em `specs/<capability>/spec.md`: ADICIONADO → anexa requisito (aborta se header já existe = conflito); MODIFICADO → substitui o bloco do requisito de mesmo header (match whitespace-insensitive); REMOVIDO → remove o bloco; RENOMEADO → renomeia o header (DE→PARA), preservando corpo.
- Conflito (header duplicado em ADICIONADO, ou MODIFICADO/REMOVIDO sem alvo existente) → exit 1 com mensagem clara, sem aplicação parcial (idempotência: aplicar num arquivo temporário e só promover no fim).
- Sucesso: move `specs/_changes/<slug>/` → `specs/_archive/AAAA-MM-DD-<slug>/` (data via `date +%F`); falha se o destino já existir (não sobrescrever histórico). Imprime resumo `+ N adicionados / ~ N modificados / - N removidos / → N renomeados`.
- `--dry-run` mostra o que faria sem escrever. `--yes` pula confirmação (uso por CI/skill).
