  <name>Task 1: Inspect all 15 agent files and capture raw audit data</name>
  <files>docs/v3/agents-audit.md (scratch/notes section — finalized in Task 2)</files>
  <action>
    Read every file in `source/agents/*.md` (15 files). For EACH agent, record, by inspecting the real file (do NOT trust the summary table — verify):
    1. **Role** — one sentence: what is this agent for? Is the description frontmatter unambiguous, or vague/overlapping with another agent?
    2. **Model routing** — what `model:` is declared (or NONE)? Is it justified by the work?
       - Heuristic: read-only search/navigation → haiku; structured code work with Edit/Bash (reviewers, fixers, refactors) → sonnet; high-stakes reasoning (security, silent-failure, planning) → opus.
       - Flag every mismatch with a concrete recommendation (e.g. "react-reviewer is sonnet, read-only — correct" / "agent X declares opus but is a mechanical refactor — downgrade to sonnet").
       - Flag agents with NO `model:` as a real gap (claude-continuation, ideiaos-checker) — state what they default to and what they SHOULD declare.
    3. **Tools** — are the declared tools minimal for the role? Flag over-grants (e.g. a reviewer with Edit/Bash it never needs) and missing tools (e.g. a fixer with no Edit).
    4. **When to use / when NOT to use** — extract or infer two short bullets each. NOT-to-use must be concrete (e.g. "do NOT use code-explorer to edit — it has no Edit").
    5. **Step-by-step directedness** — is the agent body a directed procedure an executor can follow, or vague prose? Score directedness High / Medium / Low and cite the weakest part.

    Cross-check against STATE.md known facts (e.g. `model` field added to agent schema in 04-04; mgrep/LSP candidates). Verify every claim against the actual file — this is an honest audit, include REAL weaknesses found.
