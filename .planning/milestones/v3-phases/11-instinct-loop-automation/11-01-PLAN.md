Fechar o loop de instincts automaticamente: o Stop hook `observe-session-end.sh` ganha gate de threshold + spawn de `claude -p` haiku em background quando há observações não-analisadas. `/instinct-status` ganha indicador de pendência.

Purpose: Hoje a captura é automática mas a destilação é manual. Após esta fase, o ciclo fecha sozinho — session_end registrado + gate passa + haiku roda em background + instincts atualizados.

Design decision (trigger = option b — spawn headless):
- Option (a) flag file: ainda exige ação manual na próxima sessão. Descartada.
- Option (b) claude -p haiku background no Stop hook: correto — Stop hook é o único boundary de sessão disponível. Safeguards obrigatórios: timeout 120s, nohup + disown, stdout/stderr para log, hook exit 0 imediato após spawn (sem wait). Sem recursão: claude -p headless não dispara hooks PostToolUse.
- Option (c) LaunchAgent diário: não responde à cadência de sessão, adiciona infraestrutura extra.

Gate de threshold: timestamp, não contagem de linhas. Arquivo sentinela: ~/.ideiaos/instincts/.last-analyzed-<proj> (contém ISO timestamp da última análise bem-sucedida). Se ts da última linha de observations.jsonl > last-analyzed → dispara. Dedup/compactação: NÃO truncar observations.jsonl após análise (dados históricos têm valor para reforço de confidence). Atualizar apenas o sentinela.

Lógica do gate (inserir após o bloco de append do observations.jsonl):

PROJ e OBS_DIR já estão definidos no script original. O gate precisa:

1. Ler o timestamp da última observação em observations.jsonl: usar python3 para ler a última linha não-vazia do arquivo, extrair campo "ts". Se falhar, exit 0 silencioso.

2. Ler o timestamp do sentinela ~/.ideiaos/instincts/.last-analyzed-$PROJ: se o arquivo não existir, tratar como epoch (análise nunca rodou → gate passa sempre).

3. Comparar: se ts_ultima_obs <= ts_sentinela → nada a fazer, exit 0. Se ts_ultima_obs > ts_sentinela → gate passa.

4. Quando gate passa: spawnar haiku em background com:
   - Comando: `nohup timeout 120 claude -p "$(cat ~/.ideiaos/instinct-analyze-prompt.md 2>/dev/null || echo '/instinct-analyze')" --model claude-haiku-4-5 2>>"$HOME/.ideiaos/logs/instinct-analyze-$(date +%Y%m%d-%H%M%S).log" >/dev/null`
   - Alternativa mais robusta (preferida): escrever um arquivo de prompt temporário e passar via stdin para `claude -p`, ou simplesmente passar o texto do skill inline como string. Usar a forma mais simples que funcione com a versão atual do claude CLI disponível no PATH.
   - O spawn deve ser: `( nohup timeout 120 claude --model claude-haiku-4-5 -p "/instinct-analyze" >> "$LOG_FILE" 2>&1 ) &` seguido de `disown $! 2>/dev/null || true`.
   - LOG_FILE: `$HOME/.ideiaos/logs/instinct-analyze-$(date +%Y%m%d-%H%M%S).log`. Criar dir com `mkdir -p "$HOME/.ideiaos/logs" 2>/dev/null || true` antes do spawn.
   - KILL-SWITCH: o `timeout 120` garante que o processo não ultrapasse 2 minutos. Se o claude -p travar, morre automaticamente.
   - NÃO usar `wait` — hook deve retornar imediatamente após o disown.

5. Após spawn bem-sucedido (ou falha silenciosa), NÃO atualizar o sentinela aqui. O sentinela .last-analyzed-$PROJ deve ser atualizado PELO PRÓPRIO instinct-analyze ao concluir (ver instinct-analyze/SKILL.md Passo 7 — adicionar instrução lá no plano 11-02).    Nota de implementação: como o spawn é fire-and-forget, não sabemos se instinct-analyze concluiu com sucesso. Estratégia conservadora: o sentinela é atualizado pelo instinct-analyze no final de sua execução. Se o spawn falhar (claude não instalado, timeout), o sentinela não é atualizado, o gate passará novamente na próxima sessão — comportamento correto (retry automático).

Todo o bloco de gate + spawn deve ser envolvido em `( ... ) 2>/dev/null || true` para garantir fail-silent: se claude não estiver no PATH, se observations.jsonl estiver corrompido, qualquer falha → exit 0 silencioso, nunca bloquear a sessão do usuário.

Exemplo de estrutura do bloco a acrescentar (adaptar conforme o script real):

```
# --- INSTINCT-ANALYZE AUTO-TRIGGER (R3-08 / R3-09) ---
# Gate: só dispara se há observacoes mais recentes que a ultima analise.
# Fail-silent: todo o bloco em subshell, nunca bloqueia a sessao.
# Kill-switch: timeout 120s no claude -p (sem timeout = risco OpenClaw).
(
  LAST_ANALYZED_FILE="$HOME/.ideiaos/instincts/.last-analyzed-${PROJ}"
  LOG_FILE="$HOME/.ideiaos/logs/instinct-analyze-$(date +%Y%m%d-%H%M%S).log"
  mkdir -p "$HOME/.ideiaos/logs" 2>/dev/null || exit 0

  # Extrair ts da ultima observacao
  TS_OBS="$(/usr/bin/python3 -c '
import sys, json
lines = open(sys.argv[1]).read().strip().splitlines()
for l in reversed(lines):
    l = l.strip()
    if not l: continue
    try:
        d = json.loads(l)
        ts = d.get("ts","")
        if ts: print(ts); break
    except: pass
' "$OBS_DIR/observations.jsonl" 2>/dev/null || echo "")"

  [ -z "$TS_OBS" ] && exit 0

  # Ler sentinela (epoch se ausente)
  TS_LAST="$(cat "$LAST_ANALYZED_FILE" 2>/dev/null || echo "1970-01-01T00:00:00")"

  # Gate: comparar strings ISO (lexicograficamente corretas para timestamps sem tz)
  [ "$TS_OBS" \<= "$TS_LAST" ] && exit 0

  # Gate passou: spawn haiku background com timeout
  nohup timeout 120 claude --model claude-haiku-4-5 -p "/instinct-analyze" \
    >> "$LOG_FILE" 2>&1 &
  disown $! 2>/dev/null || true
) 2>/dev/null || true
```

IMPORTANTE: o bloco de exemplo acima usa `exit 0` dentro de subshell `( )` — isso sai apenas da subshell, não do script pai. Isso é intencional e correto para fail-silent.
